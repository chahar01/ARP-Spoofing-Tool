About & Protection Methods
ARP Protocol (RFC 826)

Address Resolution Protocol maps IP addresses to MAC addresses. ARP requests are broadcast; replies can be spoofed.

How ARP Spoofing Works

An attacker sends forged ARP replies, associating their MAC with the gateway's IP. Victims send traffic to the attacker, who can intercept, modify, or drop it.

Detection Techniques
  • MAC address changes for an IP
  • Unsolicited ARP replies
  • Excessive ARP traffic
  • Duplicate IP addresses
Protection / Mitigation
  • Static ARP entries – manually set, but not scalable.
  • DHCP Snooping – switches track DHCP assignments.
  • Dynamic ARP Inspection (DAI) – switches validate ARP packets against a trusted database (DHCP snooping). Blocks invalid ARP replies.
  • Port Security – limits MAC addresses per port.
  • 802.1X – network access control.

DAI is the most effective defense because it prevents forged ARP packets from being forwarded, stopping the attack at the switch level.

References: RFC 826, SANS Network Forensics Papers, IEEE 802.1X
